# CLAUDE.md

Playwright + TypeScript E2E test suite for PDF Expert Copilot (https://copilot.pdfexpert.com/app), covering UI flows and REST API.

## Stack

- **Playwright** `^1.60.0` — UI and API testing
- **TypeScript** `^6.0.0` — strict mode, `noUnusedLocals/Parameters`
- **Zod** `^4.4.3` — API response schema validation
- **@faker-js/faker** `^10.5.0` — locale-aware fake data generation
- **imapflow + mailparser** — Gmail IMAP polling for magic-link auth
- **Husky + lint-staged** — pre-commit: `eslint --fix` + `prettier --write`

## Environment setup

Create `.env` in the project root:

```
EMAIL_USER=<gmail address>
EMAIL_PASS=<16-char Gmail App Password — NOT the account password>
TEST_ENV=prod
```

Auth requires Gmail 2FA. Go to myaccount.google.com → Security → 2-Step Verification → App passwords.  
CI secrets: `Settings → Secrets and variables → Actions`.

## Commands

```bash
npm run test:ui               # all UI tests (chromium + firefox)
npm run test:ui:chromium      # Chromium only
npm run test:ui:sanity        # --grep @sanity
npm run test:ui:copilot       # --grep @copilot
npm run test:api              # API tests via API/playwright.api.config.ts

npm run check                 # tsc + eslint + prettier (run before every PR)
npm run lint:fix              # auto-fix lint errors
npm run format                # prettier write
```

## Architecture

```
POM/              Page Object classes — all extend BasePage; locators are private fields
  Pages.ts        Aggregate entry point: one field per page/popup, passed as `app` fixture
  Popups/         Popup-scoped page objects
tests/            Spec files + auth.setup.ts (login → playwright/.auth/user.json)
  seed.spec.ts    Seed scaffold used by the MCP Playwright planner to prepare the test env
helpers/          EmailHelper (IMAP magic-link poller), FakePerson, constants
API/
  API_tests/      API specs — use RequestHandler builder + Zod assertion pattern
  utils/          RequestHandler, APILogger, fixtures.ts, ZOD_JSON_schemas/
  playwright.api.config.ts   Separate config; re-uses auth.setup.ts from tests/
pdfFiles/         PDF fixture files committed to the repo
interfaces/       Shared TypeScript types (DocumentActionsMenuPoints, Languages, etc.)
specs/            Test plans generated by the MCP Playwright planner agent
```

## Path aliases (tsconfig.json)

| Alias           | Resolves to    |
| --------------- | -------------- |
| `@pages/*`      | `POM/*`        |
| `@helpers/*`    | `helpers/*`    |
| `@utils/*`      | `API/utils/*`  |
| `@interfaces/*` | `interfaces/*` |

Use aliases for all cross-directory imports — never relative `../..` paths.

## Key conventions

**Custom `test` fixture** — never import `test` directly from `@playwright/test`.  
UI specs: `import { test } from '../test-options'` → provides `app: Pages` and `apiContext`.  
API specs: `import { test } from '@utils/fixtures'` → provides `api: RequestHandler` and `config`.

**Auth flow** — `tests/auth.setup.ts` polls Gmail via IMAP for the magic link and writes state to
`playwright/.auth/user.json`. Both UI (`chromium`/`firefox`) and `api` projects depend on `setup`.

**Test isolation** — `beforeEach` must call `app.uploadFile.deleteFileIfExist()`.  
Do NOT use `test.describe.configure({ mode: 'serial' })`.

**Tags** — every test must have a tag; CI filters via `--grep`. Current tags: `@sanity`, `@copilot`,
`@MainActions`, `@api`, `@documents`.

**Locators** — prefer `getByRole()` with `name:`; CSS classes are acceptable for structural elements.  
`playwright/no-wait-for-timeout` is an **ESLint error** — use `waitFor({ state })` instead.  
`no-floating-promises` is an error — every async call must be explicitly awaited.

**API tests** — `RequestHandler` is a builder; fields reset after each call:

```ts
api.url(config.PDF_API_PROD).path('/documents/recent').GET_Request(200);
```

Validate response shape with Zod `safeParse`:

```ts
const result = schema.safeParse(data);
expect(result.success, JSON.stringify(result.error?.issues, null, 2)).toBe(true);
```

Zod schemas live in `API/utils/ZOD_JSON_schemas/` — add one per new endpoint.

**`interfaces/Languages`** — partial list of the most-used test locales (`'English' | 'Korean' | 'Italian'`);
not exhaustive — extend when a new language is needed in tests.
